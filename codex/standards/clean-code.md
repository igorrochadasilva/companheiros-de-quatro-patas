# Clean Code Standard

- Prefer small functions with one clear responsibility.
- Use descriptive names for variables, functions, and components.
- Avoid duplication; extract shared helpers to `shared/lib` or feature-level hooks/services.
- Remove dead code and commented-out code blocks.
- Use comments to explain why, not what.
- Prefer early returns to reduce nesting.
- Keep explicit TypeScript types on public APIs (props, hooks, services, handlers).
